# 常见问题 (FAQ)

## 通用问题

### Q1: 这个系统是用来做什么的？

**A**: 这是一个自动整理差旅发票的系统。它可以：
- 从邮箱自动下载发票附件
- 识别发票类型（机票、火车、打车、住宿等）
- 自动重命名文件为标准格式
- 按行程归类发票
- 自动填写差旅费报销单

### Q2: 系统支持哪些类型的发票？

**A**: 目前支持以下类型：
- ✈️ 机票（航空运输电子客票行程单）
- 🚄 火车票（12306 铁路电子客票）
- 🚖 打车费（滴滴出行等出租车/网约车发票）
- 🚐 接送机（用车行程单）
- 🏨 住宿（酒店发票）
- 🍽️ 餐饮（餐费发票）

### Q3: 需要什么样的发票格式？

**A**: 系统优先从**文件名**解析信息，请使用以下格式命名：

```
{日期}_{类型}_{起点}_{终点}_{金额}_{出差人}.pdf
```

示例：
- `2026-03-15_机票_杭州_北京_887.00_王春晖.pdf`
- `2026-02-24_2026-02-25_住宿_452.00_王春晖.pdf`

如果文件名不规范，系统会尝试 OCR 识别，但准确率可能降低。

### Q4: 系统安全吗？会泄露我的发票信息吗？

**A**:
- ✅ 所有数据都在本地处理，不上传云端
- ✅ 不收集任何个人信息
- ✅ 开源代码，可自行审查
- ⚠️ 邮箱密码存储在本地配置文件中，请注意保护

### Q5: 可以同时处理多个出差人的发票吗？

**A**: 可以。系统会根据发票上的姓名自动分类。
- 在 `config/travelers.yaml` 中配置出差人信息
- 系统会自动识别发票上的姓名并归类到对应出差人文件夹

## 安装与配置

### Q6: 必须使用浙大邮箱吗？

**A**: 不必须。任何支持 IMAP 协议的邮箱都可以：
- ✅ 浙大邮箱 (imap.zju.edu.cn)
- ✅ Gmail (imap.gmail.com)
- ✅ QQ邮箱 (imap.qq.com)
- ✅ 163邮箱 (imap.163.com)
- ✅ 企业邮箱（需确认 IMAP 地址和端口）

### Q7: 如何获取邮箱的应用专用密码？

**A**:
- **Gmail**:
  1. 开启两步验证
  2. 进入 Google 账户设置
  3. 选择"安全性" → "应用专用密码"
  4. 生成新密码

- **QQ邮箱**:
  1. 登录 QQ 邮箱网页版
  2. 设置 → 账户
  3. 开启"POP3/IMAP/SMTP/Exchange/CardDAV/CalDAV服务"
  4. 生成授权码

- **浙大邮箱**:
  - 直接使用邮箱密码（或联系管理员）

### Q8: 不想配置邮箱，可以用其他方式吗？

**A**: 可以。手动将发票放入 `invoices/` 目录，然后运行：
```bash
python main.py --run
```

系统会处理目录中的所有文件。

### Q9: Windows 任务计划程序是什么？

**A**: Windows 自带的定时任务工具。配置后可以：
- 每天自动运行程序
- 无需手动操作
- 开机自动启动

配置方法：运行 `scripts\setup_task.bat`（需要管理员权限）

## 使用问题

### Q10: 如何首次运行系统？

**A**: 三步走：
1. 运行 `install.bat` 安装依赖
2. 编辑 `config/email.yaml` 配置邮箱
3. 运行 `start.bat` 开始使用

### Q11: 如何只处理一次，不启动监控？

**A**: 双击 `scripts\manual_run.bat` 或运行：
```bash
python main.py --run
```

### Q12: 如何查看处理统计？

**A**: 运行：
```bash
python main.py --stats
```

会显示已处理的文件数、识别的发票类型等统计信息。

### Q13: 行程归类不准确怎么办？

**A**:
1. 检查城市名称是否标准
2. 检查发票日期是否正确
3. 手动调整：将发票移动到正确的行程文件夹
4. 修改 `config/parsers.yaml` 添加城市映射

### Q14: 如何生成报销单？

**A**:
1. 确保已有行程归类（`trips/` 目录下有文件夹）
2. 将报销单模板放到 `trips/报销单.doc`
3. 运行：
   ```bash
   python fill_reimbursement_template.py
   ```
4. 报销单会自动生成到每个行程文件夹

### Q15: 发票和行程单（价格相同）会被重复计算吗？

**A**: 不会。系统会自动去重：
- 同一日期、同一类型、同一金额的发票
- 优先保留"发票"而非"行程单"
- 确保每笔费用只计算一次

## 故障排查

### Q16: OCR 识别失败怎么办？

**A**:
1. 确保图片清晰（建议扫描件）
2. 使用标准文件名格式（系统会优先解析文件名）
3. 手动重命名文件为标准格式

### Q17: 程序运行缓慢怎么办？

**A**:
1. 首次运行会下载 OCR 模型（约100MB），请耐心等待
2. 大量文件处理需要时间，属正常现象
3. 可以分批处理文件

### Q18: 如何重置系统，重新开始？

**A**:
1. 删除数据库：`del data\records.db`
2. 清空处理记录：系统会重新处理所有文件
3. 或删除 `invoices/` 和 `trips/` 中的文件

### Q19: 如何卸载系统？

**A**:
1. 运行 `scripts\remove_tasks.bat` 删除定时任务
2. 删除项目目录
3. 删除环境变量（如已设置）

## 高级问题

### Q20: 可以自定义发票识别规则吗？

**A**: 可以。编辑 `config/parsers.yaml`：
- 添加新的发票类型关键词
- 自定义金额提取正则表达式
- 添加城市映射关系

### Q21: 可以对接其他系统吗？

**A**: 可以。系统提供了 Python API：
```python
from src.scheduler import TaskScheduler

scheduler = TaskScheduler()
result = scheduler.run_once()
```

也可以直接调用各个模块（`src/` 目录下）。

### Q22: 如何添加新的出差人？

**A**: 编辑 `config/travelers.yaml`：
```yaml
travelers:
  - name: "张三"
    title: "研究员"
    department: "某某实验室"
```
