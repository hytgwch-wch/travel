# 故障排查指南

## 常见问题及解决方案

### 邮箱同步问题

#### 问题：无法连接到邮箱服务器

**错误信息**: `AuthenticationFailed` 或 `Connection refused`

**可能原因**:
1. IMAP 服务未开启
2. 用户名或密码错误
3. 网络连接问题
4. 防火墙阻止

**解决方案**:
1. 检查邮箱设置，确认 IMAP 服务已开启
2. 使用应用专用密码，而非登录密码（Gmail、QQ邮箱等）
3. 确认网络连接正常
4. 检查防火墙设置，允许 Python 访问网络

**Gmail 配置示例**:
- 开启两步验证
- 生成应用专用密码
- 使用应用密码登录

**浙大邮箱配置示例**:
- IMAP 服务器: `imap.zju.edu.cn`
- 端口: `993`
- SSL: 开启

#### 问题：下载的文件不完整或损坏

**可能原因**:
1. 网络中断
2. 磁盘空间不足
3. 文件权限问题

**解决方案**:
1. 检查磁盘空间
2. 检查 `temp/` 和 `invoices/` 目录权限
3. 删除损坏的文件，重新同步

#### 问题：ZIP 文件无法解压

**错误信息**: `File not found` 或 `Bad zip file`

**解决方案**:
1. 确认 ZIP 文件完整（手动测试解压）
2. 检查是否有特殊字符或中文路径问题
3. 尝试手动解压后放入 `invoices/` 目录

### OCR 识别问题

#### 问题：OCR 识别速度慢

**原因**: PaddleOCR 首次运行会下载模型文件

**解决方案**:
1. 等待模型下载完成（约 100MB）
2. 后续运行会使用缓存，速度正常
3. 模型缓存位置: `C:\Users\用户名\.paddleocr\`

#### 问题：OCR 识别结果不准确

**可能原因**:
1. 图片质量差（模糊、倾斜）
2. 扫描分辨率低
3. 特殊字体或格式

**解决方案**:
1. 使用高清扫描件（建议 300dpi 以上）
2. 确保图片平整、无倾斜
3. 手动重命名文件（使用标准格式）

#### 问题：OCR 模块加载失败

**错误信息**: `DLL load failed` 或 `Module not found`

**解决方案**:
1. 重新安装依赖：
   ```bash
   pip uninstall paddleocr paddlepaddle
   pip install paddleocr paddlepaddle
   ```
2. 确保安装了 Visual C++ 运行库

### 发票解析问题

#### 问题：发票类型识别错误

**可能原因**:
1. OCR 识别结果不准确
2. 关键词匹配规则不完善

**解决方案**:
1. 检查 `config/parsers.yaml` 中的关键词配置
2. 添加特定发票格式的识别规则
3. 使用文件名格式解析（更可靠）

#### 问题：金额提取错误

**可能原因**:
1. 发票格式特殊
2. 正则表达式不匹配

**解决方案**:
1. 检查 `config/parsers.yaml` 中的金额提取规则
2. 添加特定格式的正则表达式
3. 系统会优先使用文件名中的金额信息

#### 问题：出差人姓名识别错误

**可能原因**:
1. OCR 识别出错
2. 姓名"工号"等占位符未过滤

**解决方案**:
1. 检查 `config/travelers.yaml` 配置
2. 系统会自动过滤"工号"等占位符
3. 使用默认出差人配置

### 行程归类问题

#### 问题：发票未正确归类到行程

**可能原因**:
1. 地理位置不连续
2. 日期间隔过大
3. 城市名称不规范

**解决方案**:
1. 检查城市名称是否标准（如"杭州东"会被识别为"杭州"）
2. 检查 `config/parsers.yaml` 中的省份映射
3. 手动调整发票日期或目的地

#### 问题：生成行程时程序崩溃

**错误信息**: `KeyError` 或 `IndexError`

**可能原因**:
1. 发票信息不完整
2. 数据库损坏

**解决方案**:
1. 检查数据库文件 `data/records.db`
2. 删除数据库重新开始：
   ```bash
   del data\records.db
   python main.py --run
   ```

### 报销单生成问题

#### 问题：.doc 文件无法处理

**错误信息**: `Package not found` 或 `Unsupported format`

**原因**: python-docx 不支持 .doc 格式

**解决方案**:
1. 安装 pywin32：
   ```bash
   pip install pywin32
   ```
2. 将 .doc 转换为 .docx 格式
3. 系统会自动使用 Word COM 接口处理 .doc 文件

#### 问题：报销单填写位置错误

**可能原因**:
1. 模板格式不同
2. 表格结构变化

**解决方案**:
1. 检查模板文件格式
2. 修改 `fill_reimbursement_template.py` 中的行列索引
3. 使用与模板一致的 Word 文档

### 性能问题

#### 问题：处理速度慢

**可能原因**:
1. OCR 识别耗时
2. 大量文件待处理
3. 硬件性能限制

**解决方案**:
1. 批量处理，分时段运行
2. 使用 GPU 加速 OCR（需安装 GPU 版 PaddlePaddle）
3. 定期清理已处理的文件

#### 问题：内存占用过高

**解决方案**:
1. 减少 OCR 并发数
2. 分批处理文件
3. 定期重启程序

### 其他问题

#### 问题：日志文件过大

**解决方案**:
1. 配置日志滚动（loguru）
2. 定期清理旧日志
3. 调整日志级别为 WARNING 或 ERROR

#### 问题：程序无响应

**解决方案**:
1. 检查是否正在处理大文件
2. 查看日志文件 `logs/` 了解状态
3. 使用任务管理器结束进程，重启程序

## 获取帮助

如果以上方案无法解决问题：

1. **查看日志**: `logs/` 目录下的日志文件
2. **运行测试**: `pytest tests/` 检查模块状态
3. **提交 Issue**: 在 GitHub 仓库提交问题，附带：
   - 错误信息
   - 相关日志
   - 系统环境（Python 版本、操作系统）

## 调试模式

启用详细日志：

```bash
# 设置环境变量
set LOGURU_LEVEL=DEBUG
python main.py --run
```

或在 `config/config.yaml` 中设置：

```yaml
logging:
  level: DEBUG
  file: "logs/debug.log"
```
