# VISION.md - 出差发票自动整理系统

## 1. 项目初衷

用户频繁出差，产生大量交通、住宿等发票（PDF/图片格式），通过手机上传至**百度网盘云端**。文件名为原始名称难以管理检索。需要构建自动化系统，从云端同步/下载发票文件到本地，识别后规范化命名并存储到本地分类目录。

## 2. 核心功能点

### 2.1 百度网盘同步（bypy 方式）

**bypy** - 百度网盘 Python 命令行客户端
- GitHub: https://github.com/houtianze/bypy
- 优点：开源免费、纯Python实现、无需官方客户端、支持命令行操作
- 功能：下载、上传、列表、同步等

**集成方式**：
```python
# 方式1：直接调用 bypy 命令行
import subprocess
subprocess.run(['bypy', 'download', '远程路径', '本地路径'])

# 方式2：使用 bypy 的 Python 库
from bypy import ByPy
bp = ByPy()
bp.download('远程路径', '本地路径')
```

### 2.2 本地扫描
- 扫描本地同步目录中的新文件
- 基于文件修改时间或SQLite记录检测增量

### 2.2 智能识别
- **发票类型分类**：
  - 交通类：机票、火车票、打车票/网约车
  - 住宿类：酒店发票
  - 其他：餐饮、汽车租赁、过路费等
- **提取关键信息**：
  - 发票类型
  - 日期
  - 金额
  - 出差人
  - **交通类专属**：起始地、目的地、交通方式
  - **住宿类专属**：城市、酒店名称、入住天数

### 2.3 规范化重命名

**命名规则**：

| 发票类型 | 命名格式 | 示例 |
|---------|---------|------|
| 机票 | `{日期}_机票_{出发地}_{目的地}_{金额}_{出差人}.pdf` | `2025-02-20_机票_北京_上海_850.00_张三.pdf` |
| 火车票 | `{日期}_火车_{出发地}_{目的地}_{金额}_{出差人}.pdf` | `2025-02-20_火车_南京_杭州_125.50_张三.pdf` |
| 打车/网约车 | `{日期}_打车_{出发地}_{目的地}_{金额}_{出差人}.pdf` | `2025-02-20_打车_上海站_希尔顿酒店_45.00_张三.pdf` |
| 住宿 | `{日期}_住宿_{城市}_{酒店名}_{金额}_{出差人}.pdf` | `2025-02-21_住宿_上海_希尔顿酒店_480.00_张三.pdf` |
| 餐饮 | `{日期}_餐饮_{城市}_{金额}_{出差人}.pdf` | `2025-02-20_餐饮_上海_128.00_张三.pdf` |
| 汽车租赁 | `{日期}_租车_{城市}_{天数}天_{金额}_{出差人}.pdf` | `2025-02-20_租车_深圳_3天_450.00_张三.pdf` |
| 其他 | `{日期}_其他_{类型}_{金额}_{出差人}.pdf` | `2025-02-20_其他_办公用品_35.00_张三.pdf` |

### 2.4 分类存储（本地）
```
本地整理目录/
├── 2025/
│   ├── 02月/
│   │   ├── 交通/
│   │   ├── 住宿/
│   │   ├── 餐饮/
│   │   └── 其他/
```

**数据流**：
```
百度网盘云端 → 本地同步目录 → OCR识别 → 本地整理目录
```

### 2.5 源文件处理策略
- **保留原文件**：云端文件保持不变
- **处理标记**：在SQLite中记录已处理文件（避免重复处理）
- **可选**：处理完成后删除本地同步目录中的文件（节省空间）

## 4. 技术可行性评估

### 4.1 OCR技术选型（免费方案）

| 方案 | 成本 | 中文支持 | 发票识别 | 推荐度 |
|-----|------|---------|---------|--------|
| **PaddleOCR** | 完全免费 | 优秀 | 通用OCR + 自定义解析 | ⭐⭐⭐⭐⭐ |
| Tesseract | 免费 | 较差 | ❌ | ⭐⭐ |
| EasyOCR | 免费 | 良好 | 通用OCR | ⭐⭐⭐ |
| 百度OCR API | 按量付费 | 优秀 | 专用接口 | ⭐⭐⭐ |

**决策**：使用 **PaddleOCR**（完全免费、本地运行、中文识别优秀）

**发票信息解析策略**：
- 使用正则表达式从OCR结果中提取关键字段
- 针对不同发票类型编写专用解析模板
- 支持自定义解析规则（配置文件）

### 4.2 特殊发票识别要点

| 发票类型 | 关键识别特征 | 提取字段 |
|---------|------------|----------|
| 机票/行程单 | "航空运输电子客票行程单"、航班号 | 出发地、目的地、日期、金额、姓名 |
| 火车票 | "中国铁路"、车次、座位号 | 出发站、到达站、日期、金额、姓名 |
| 打车票 | "出租车"、"网约车"、里程 | 起点、终点、日期、金额 |
| 增值税发票 | "发票号码"、"开票日期"、购买方 | 销售方、日期、金额 |

### 4.3 定时任务方案
- Windows: Task Scheduler + Python 脚本
- Python: `apscheduler` 库

### 4.4 文件监控方案
- 扫描本地同步目录 + SQLite 记录已处理文件
- 备选: `watchdog` 库实时监控本地同步目录

### 4.5 百度网盘对接（bypy）

| 技术点 | 说明 |
|-------|------|
| 工具 | bypy（百度网盘Python CLI） |
| 安装 | `pip install bypy` |
| 认证 | 首次运行需浏览器授权（OAuth） |
| 核心命令 | `bypy list`、`bypy download`、`bypy syncdown` |
| 工作目录 | bypy默认操作 `/apps/bypy/` 目录 |

**bypy 目录结构**：
```
百度网盘/
└── apps/
    └── bypy/           # bypy 的工作目录
        └── invoices/   # 发票原始文件存放处
            ├── 机票_001.pdf
            ├── 酒店_002.jpg
            └── ...
```

**同步策略**：
1. 定时调用 `bypy list` 获取远程文件列表
2. 与本地数据库比对，识别新增文件
3. 调用 `bypy download` 下载新文件到临时目录
4. OCR识别并处理后移动到本地整理目录

## 5. 风险预警

| 风险 | 影响 | 缓解措施 |
|-----|------|----------|
| 百度网盘同步延迟 | 新文件未及时处理 | 检查同步状态，支持手动触发 |
| 同步目录占用空间大 | 磁盘空间不足 | 处理后自动删除本地副本 |
| OCR识别率不足 | 信息提取不准确 | 多模板 + 人工校验机制 |
| 复杂发票无法识别 | 文件无法重命名 | 保留原始文件名，记录待处理列表 |
| 文件名冲突 | 覆盖已有文件 | 检测冲突，添加时间戳后缀 |
| 出差人识别失败 | 命名缺失 | 配置默认出差人 + 交互式询问 |
| 发票类型误判 | 分类错误 | 人工审核入口 + 重新分类功能 |
| 网络断开 | 无法获取新文件 | 离线模式支持，等待恢复后继续 |

## 7. 配置文件结构

```yaml
# config.yaml
# 百度网盘配置（bypy）
baidu_pan:
  remote_dir: "invoices"           # 远程目录（/apps/bypy/下）
  temp_dir: "temp"                 # 本地临时下载目录
  process_after_download: true     # 下载后自动处理

# 本地整理目录（输出）
local_output_dir: "invoices"

# 默认出差人
default_traveler: "张三"

# 处理选项
options:
  delete_temp_after_process: true  # 处理后删除临时文件
  skip_existing: true              # 跳过已处理的文件
  dry_run: false                   # 模拟运行（不实际移动文件）
```
